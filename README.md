## 🗂️ Project Structure & File Breakdown

### 🖥️ Core Application (`src/`)
*   **[detector.py](src/detector.py)**: The main engine. It wraps the YOLOv11 model, handles the inference logic, and coordinates between vehicle detection and license plate extraction.
*   **[plate_detector.py](src/plate_detector.py)**: A specialized module that focuses on license plates. It uses YOLO to find the plate and **EasyOCR** to read the text.
*   **[preprocessor.py](src/preprocessor.py)**: Handles image enhancement (CLAHE, denoising) to improve detection accuracy in low-light or blurry conditions.
*   **[utils.py](src/utils.py)**: Helper functions for logging, file I/O, and drawing bounding boxes.

### 🌐 User Interface (`app/`)
*   **[streamlit_app.py](app/streamlit_app.py)**: The web-based dashboard. Allows users to upload images or videos and visualize real-time ANPR results.

### 🛠️ ML Utility Scripts (`scripts/`)
*   **`auto_label.py` / `auto_label_plates.py`**: Automated labeling tools that use AI to pre-annotate your dataset, saving hours of manual work.
*   **`train_from_scratch.py`**: Script to initiate model training on your local machine using the configured dataset.
*   **`optimize.py`**: Converts `.pt` models to **ONNX** or **TensorRT** and applies quantization for faster performance on edge hardware.
*   **`evaluate.py`**: Generates precision/recall metrics and mAP scores to verify model accuracy.
*   **`check_dataset.py`**: Scans your data folder to ensure every image has a valid label file and correct formatting.

### ⚙️ Configuration & Entry
*   **[vehicle_detection.yaml](configs/vehicle_detection.yaml)**: The "brain" of the project settings. Control everything from confidence thresholds to training hyperparameters here.
*   **[main.py](main.py)**: The Command Line Interface (CLI) entry point. Use this to run detections from the terminal without the web UI.
*   **[requirements.txt](requirements.txt)**: Lists all necessary Python libraries (Ultralytics, OpenCV, Streamlit, etc.).

### 📂 Workspace Directories
*   **`models/`**: Stores your trained brain (the `.pt` weights).
*   **`runs/`**: Automatic logs generated by YOLO during training (plots, weights, checkpoints).
*   **`outputs/`**: Where the app saves your results (annotated images/videos and JSON data).
*   **`scratch/`**: Your personal sandbox for testing new code snippets or one-off debugging scripts.

## 🚀 Quick Start

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Run the Streamlit UI
```bash
streamlit run app/streamlit_app.py
```

### 3. CLI Inference
```bash
# Detect on an image
python main.py --source data/raw/sample.jpg --save

# Run with full metrics benchmark
python main.py --source data/raw/traffic.mp4 --metrics
```
